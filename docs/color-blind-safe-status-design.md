# ThreadBeacon 色盲安全状态标识设计

## 状态

- 设计日期：2026-07-21
- 范围：macOS ThreadBeacon 主任务列表、Settings 和数据源健康入口一致性验证
- 默认行为：关闭，不改变现有用户界面

## 问题

ThreadBeacon 当前主任务状态使用不同颜色的圆点，同时在下一行显示状态文字。文字已经提供第二
通道，但用户扫视列表时仍可能先依赖颜色；红、黄、绿等颜色对部分色觉差异用户不够稳定。

数据源健康入口已经使用盾牌、三角形和八边形区分健康、降级与不可用，因此本次重点补齐主任务
状态灯，不重做状态推导、排序、提示音或健康数据模型。

## 方案比较

### 方案 A：仅更换为色盲友好调色板

优点是改动最小。缺点是不同色觉类型没有一套通用颜色组合，仍然只依赖颜色，不满足核心目标。

### 方案 B：可选的颜色、形状和文字三通道标识

默认保持现有圆点；用户开启“色盲安全状态标识”后，每种任务状态使用独立 SF Symbol，继续保留
语义颜色和状态文字。该方案不需要判断用户的色觉类型，也不会改变未开启用户的熟悉界面。

### 方案 C：所有用户始终使用独立图标

一致性最好，但会直接改变现有紧凑列表的视觉密度，不适合作为当前版本的默认迁移。

采用方案 B。

## 设置与持久化

- 在 Settings 的“外观”Section 增加 Toggle：“色盲安全状态标识”。
- 辅助说明：“同时使用颜色、形状和文字区分任务状态。”
- 偏好键：`displayColorBlindSafeStatusIndicators`。
- 默认值：`false`。
- 修改后立即生效，不需要刷新数据或重启 App。
- `DisplaySettings` 和 `DisplaySettingsRepository` 保存该字段，保持设置模型的单一事实来源。

## 视觉映射

| 状态 | 标准模式 | 色盲安全模式 SF Symbol | 文字 |
| --- | --- | --- | --- |
| 错误 | 红色圆点 | `xmark.octagon.fill` | 错误／服务失败 |
| 需要操作 | 橙色圆点 | `exclamationmark.square.fill` | 需要操作 |
| 服务异常 | 黄色圆点 | `exclamationmark.triangle.fill` | 服务异常 |
| 运行中 | 蓝色圆点和外圈 | `play.circle.fill` | 运行中 |
| 刚完成 | 绿色圆点 | `checkmark.circle.fill` | 刚完成 |
| 空闲 | 次要色圆点 | `minus.circle.fill` | 空闲 |
| 未知 | 黄色圆点 | `questionmark.circle.fill` | 未知 |

所有图标固定在 `18 x 18` 布局槽内，视觉主体控制在约 `12 x 12`，不得改变任务标题、Token、
持续时间或 Subagent 列宽。运行中图标不新增连续动画，避免小窗口中产生额外视觉噪声。

## 交互与无障碍

- 状态标识使用本地化状态名称作为 Tooltip。
- 状态图形对 VoiceOver 保持隐藏，因为同一行已有可读状态文字，避免重复朗读。
- 开关使用原生 Toggle，不增加自定义模式选择器。
- 色盲安全模式只改变视觉呈现，不改变状态优先级、提示音、自动恢复或刷新逻辑。
- 底部数据源健康入口继续使用现有盾牌、三角形和八边形，保持颜色、形状、Tooltip 和
  Accessibility Label 四个通道。

## 国际化

新增中文源字符串和英文翻译：

- 色盲安全状态标识 / Color-blind-safe status indicators
- 同时使用颜色、形状和文字区分任务状态。 /
  Distinguish task status with color, shape, and text.

状态名称复用现有本地化字符串。

## 测试

### 自动测试

- `DisplaySettings` 默认关闭色盲安全模式。
- Repository 可以保存并重新加载开关。
- 新的状态视觉语义模型为七种 `ThreadDisplayStatus` 返回唯一且稳定的 symbol。
- 标准模式与安全模式都保留稳定的 `18 x 18` 外部尺寸契约。
- 现有完整测试、Debug build 和 Release build 继续通过。

### UI 验收

- Settings 开关在简体中文和英文下完整显示，修改后主列表立即变化。
- 标准模式保持当前圆点外观。
- 安全模式下七种状态不依赖颜色即可从形状区分。
- 浅色、深色模式均保持足够对比度。
- 最小窗口宽度、不同任务数量、Subagent 收起／展开时不改变标题和 Token 列宽。
- 底部健康入口在健康、降级和不可用状态下继续以不同形状显示。

## 非目标

- 不模拟医学意义上的色觉滤镜。
- 不新增多套色盲类型配置。
- 不改变品牌图标或 App 图标中的红黄绿灯。
- 不改变截图以外的通知声音、服务异常判断和数据读取。
