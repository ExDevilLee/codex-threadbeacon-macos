# 界面国际化设计

## 目标

ThreadBeacon 首批支持简体中文与英文。Settings 提供`跟随系统`、`简体中文`和`English`
三档选择，切换后立即影响主窗口与 Settings，不要求重启 App。

系统语言解析规则：

- 中文语言环境使用简体中文。
- 英文语言环境使用英文。
- 其他尚未支持的系统语言回退到英文。
- 用户显式选择的语言优先于系统语言。

后续会继续增加更多语言；英文始终作为未支持语言的最终回退。

## 方案选择

采用 Apple 标准本地化资源和 SwiftUI `Locale` 环境覆盖：

1. `AppLanguage` 只保存语言偏好和解析规则，不保存界面文案。
2. `Localizable.xcstrings` 保存简体中文与英文翻译。
3. App 根节点根据设置注入 `Locale`，语言选择变化时 SwiftUI 立即重绘。
4. Core 中的状态、计数和时间值继续保持语义数据；界面层将其映射为可本地化文本。
5. 原始数据库或系统错误保留诊断信息，稳定的错误标题和操作提示由界面层本地化。

不采用修改全局 `AppleLanguages` 后重启的方案，因为切换反馈慢且会影响进程级设置；也不
维护自定义中英文字典，避免偏离 macOS 标准本地化工具链。

## Settings 交互

`通用`页新增`语言`区块，Picker 选项为：

- `跟随系统`
- `简体中文`
- `English`

选择通过 `UserDefaults` 持久化。首次启动与旧版本升级默认使用`跟随系统`，不迁移或覆盖
其他现有 Settings。

## 文案覆盖范围

首版覆盖：

- 主窗口标题、任务计数、空状态、按钮 Tooltip 和无障碍标签。
- 状态名称、运行时长、重试信息和归档/收藏/置顶菜单。
- Subagent 展开、详情和 Token 详情。
- 数据源健康详情。
- Settings 通用、提示音和登录启动状态。
- 用户可操作的 Alert、Confirmation Dialog 和稳定错误标题。

任务标题、Agent 名称、模型名、HTTP 状态码和文件/产品名保持原值，不做翻译。

## 构建与测试

- Xcode App target 编译 `Localizable.xcstrings`。
- SwiftPM Core 测试增加语言解析规则测试，不依赖 Xcode UI。
- Xcode Debug 构建验证 string catalog 可编译并进入 App bundle。
- 运行时分别验证简体中文、英文和“非中英文系统语言回退英文”。
- 检查浅色/深色和最小窗口宽度不因英文文案明显溢出。

## 边界

- 本阶段不自动翻译用户任务标题和日志内容。
- 本阶段不引入第三方本地化框架。
- 后续新增语言时扩展 `AppLanguage`、string catalog 和截图，不改变回退规则。
